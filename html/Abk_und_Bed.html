Seite 48		          
  INIT               Dieser Ereigniseingang muss auf ein Anfragetelegramm abgebildet werden, das eine Initialisierung des Dienstes anfordert, der durch die Funktionsbausteininstanz zur Verfügung gestellt wird, z. B. die lokale Initialisierung einer Kommunikationsverbindung oder eines Prozessschnittstellenmoduls. This event input shall be mapped to a request primitive which requests an initialization of the service provided by
the function block instance, e.g., local initialization of a communication connection or a process interface module.
  INITO               Dieser Ereignisausgang muss auf ein Bestätigungstelegramm abgebildet werden, das die Beendigung einer Prozedur einer Dienstinitialisierung anzeigt. This event output shall be mapped to a confirm primitive which indicates completion of a service initialization
procedure

  QI: BOOL               Dieser Eingang stellt ein Besti Beispiel beim Auftreten eines Bestimmungszeichen an den Telegrammen dar, die auf die Ereigniseingänge abgebildet sind. Wenn zum reignisses INIT dieser Eingang TRUE ist, wird die Initialisierung des Dienstes angefordert; wenn er FALSE ist, wird die Beendigung des Dienstes angefordert. This input represents a qualifier on the service primitives mapped to the event inputs. For instance, if this input is
TRUE upon the occurrence of an INIT event, initialization of the service is requested; if it is FALSE, termination of
the service is requested.
  QO: BOOL               Diese Variable stellt ein Bestimmungszeichen an den Telegrammen dar, die auf die Ereignisausgänge abgebildet werden. Zum Beispiel zeigt ein Wert TRUE an diesem Ausgang beim Auftreten eines Ereignisses INITO die erfolgreiche Initialisierung des Dienstes an; ein Wert FALSE zeigt an, dass die Initialisierung nicht erfolgreich war. This variable represents a qualifier on the service primitives mapped to the event outputs. For instance, a TRUE
value of this output upon the occurrence of an INITO event indicates successful initialization of the service; a
FALSE value indicates unsuccessful initialization.
  PARAMS               Dieser Eingang enthält einen oder mehrere Parameter, die dem Dienst zugeordnet sind, typischerweise als Elemente eines strukturierten Datentyps. Wenn dieser Eingang vorhanden ist, muss die Festlegung des Funktionsbausteintyps seine Datentypen und voreingestellte Anfangswerte definieren. Eine Typfestlegung des Dienstschnittstellen-Funktionsbausteins darf einen oder mehrere Dienstparametereingänge für diesen Eingang ersetzen. This input contains one or more parameters associated with the service, typically as elements of an instance of a
structured data type. When this input is present, the function block type specification shall define its data type and default initial value(s).
A service interface function block type specification may substitute one or more service parameter inputs for this
input.
  STATUS               Dieser Ausgang muss einen Datentyp besitzen, der geeignet ist, den Status des Dienstes beim Auftreten eines Ereignisausgangs
auszudrücken.
Eine Dienstfestlegung darf anzeigen, dass der Wert dieses Ausgangs in einigen Situationen nicht relevant ist; z. B. bei INITO+ IND+ und CNF+, so wie es in 6.1.3 beschrieben ist.		 This output shall be of a data type appropriate to express the status of the service upon the occurrence of an event
output.
A service specification may indicate that the value of this output is irrelevant for some situations,for instance, for
INITO+, IND+ and CNF+ as described in 6.1.3.
